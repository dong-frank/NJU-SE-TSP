# 典型TSP角色

1. 项目组长
2. 计划经理：第四次会议和第六次会议的组织者，计划是大家讨论出来的，不是计划经理决定，计划经理带领团队开发完整的计划

    1. 第四次会议：带领项目小组开发项目计划
    2. 第六次会议：带领项目小组平衡计划
3. 开发经理：

    1. 第三次会议的开发策略，通常由开发经理确定
    2. 第四次会议带领团队做估算
4. 质量经理：提升质量不是通过测试，而是通过评审。向项目组长警示质量问题

    1. 第五次会议的主持者
5. 过程经理：过程的质量决定了结果的质量

    1. 第三次会议由过程经理带领定义过程
6. 支持经理：项目小组在整个开发过程中都有合适的工具和环境

    1. 不负责任何一次会议，但会支持每一次会议
7. 开发人员: 所有人员都是开发人员
